题目：实现一个微信移动端活动页面。

解答：Demo

一些说明：

  • 简述：适配各种尺寸的手机机型，动画方面在 Chrome 移动模式下能够完全正常显示。
  • 耗时：由于自己时间安排上的冲突，实际是从8月27日下午开始做，直到8月28日晚。
  • JS：使用了 swiper.js 这个插件。
  • 动画：使用了 Animate.css 这个 CSS3 动效库。
  • 性能：常规的 CSS/JS 文件压缩合并；图片使用七牛提供的 CDN 功能。
  • 问题：iPhone6 真机打开时，首页动画不显示；所有使用 bounceInDown 这个动画效果的元素在 iPhone6 真机上不能正常显示。以上问题目前正在解决中。
  • 改进：虽然用了 CDN 之后，加载速度有提升，不过还可以考虑给除了首屏外的其他图片进行 CSS sprite 合并；五星档案页(黄色那页)里面的折线处可以考虑使用 SVG 制作动画；swiper.min.js 有 67kb，还是有点大，可以抽离出自己需要使用的部分，或者自己之后写一个更加轻量的；「更多评语」和「X」两处按钮可以做一个小的按键动画。
  • 其他：PSD 文件里面有些字体丢失，没有去找回(因为感觉不是答题的重点，当然实际工作中肯定是会去找回的)；呃...其实是第一次制作移动端的这种宣传页，所以还有不少做得不够好的地方，还望前辈指点。
